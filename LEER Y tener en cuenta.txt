COMPARACIÓN DOMINIO 2 — ERD vs CÓDIGO DEL PROYECTO
====================================================

LO QUE ESTÁ BIEN ✓
-------------------

- Categoria       → Correcta: todos los campos, auto-referencia parent_id, audit fields
- ProductoCategoria → Correcta: PK compuesta, es_principal, created_at
- Producto (parcial) → Campos base correctos: precio_base, imagenes_url, stock_cantidad,
                       disponible, audit fields


PROBLEMAS ENCONTRADOS ✗
------------------------

1. Entidad "UnidadMedida" FALTA COMPLETAMENTE
   El ERD define esta tabla de catálogo con:
     - nombre    : VARCHAR(50)  {UQ, NN}
     - simbolo   : VARCHAR(10)  {UQ, NN}
     - tipo      : VARCHAR(20)  {NN}
     - created_at: TIMESTAMPTZ  {NN}
   No existe ningún módulo unidad_medida/ en el proyecto.

2. "Producto" le falta el FK unidad_venta_id
   El ERD especifica:
     + unidad_venta_id : BIGINT {FK → UnidadMedida.id, NULL}
   En app/modules/producto/model.py ese campo no existe.

3. "ProductoIngrediente" le faltan 2 campos
   El ERD define:
     + cantidad         : DECIMAL(10,3)  {NN, CHECK > 0}
     + unidad_medida_id : BIGINT         {FK → UnidadMedida.id, NN}
   En app/modules/producto/link_models.py solo existe es_removible,
   sin cantidad ni unidad_medida_id.

4. "Ingrediente" tiene deleted_at DE MÁS
   El modelo tiene deleted_at (soft-delete), pero el ERD NO lo incluye
   para Ingrediente — solo created_at y updated_at.
   Ese campo sobra según el diseño.


RESUMEN DE PRIORIDAD
---------------------

  ALTO   → UnidadMedida falta: rompe toda la lógica de "precio por kg/unidad"
  ALTO   → Producto.unidad_venta_id falta: depende de UnidadMedida
  MEDIO  → ProductoIngrediente.cantidad + unidad_medida_id faltan:
           sin estos no se puede saber cuánto de cada ingrediente lleva un producto
  BAJO   → Ingrediente.deleted_at de más: campo extra que no estaba en el diseño
